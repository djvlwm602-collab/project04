# 프로젝트 기획서: Bye-Company Vibe (파이어족 은퇴 D-Day 시뮬레이터)

## 1. 프로젝트 개요
- **서비스명:** Bye-Company Vibe
- **타겟 유저:** 40대 맞벌이 딩크족(DINK) 직장인
- **목표:** "언제 회사를 당당하게 그만둘 수 있는가?"를 시각적으로 보여주는 은퇴 타이머 및 흥미로운 자산 관리 웹 서비스

## 2. 디자인 접근 전략 (카카오뱅크 스타일)
- **핵심 키워드:** 직관성, 친근함, 큼직한 타이포그래피, 깔끔한 여백
- **컬러 팔레트:**
  - **Primary:** 선명하고 상쾌한 옐로우 (친근함 강조)
  - **Background:** 깔끔한 화이트(기본) & 몰입감을 줄 수 있는 다크 모드
  - **Point:** 알림이나 위기 상황에는 명확한 블루/레드 톤 사용
- **타이포그래피:** 굵고 가독성 높은 폰트로 '남은 시간'과 '목표 금액'을 가장 눈에 띄게 배치
- **컴포넌트:** 둥근 모서리(Border-radius)를 넉넉히 적용한 카드 형태, 부드러운 스와이프 UI 애니메이션

## 3. 핵심 기능 (Core Features)

### 3.1. 카카오/구글 소셜 로그인
- 가장 빠르고 스트레스 없는 원클릭 로그인 연동
- **구글 연동:** 은퇴 D-Day를 구글 캘린더에 자동 기록
- **카카오 연동:** 부부끼리 은퇴일 당기기 또는 지출 참기 내역을 카카오톡으로 쉽고 예쁘게 공유

### 3.2. 은퇴 D-Day 대시보드 (메인 화면)
- **통계 자동화:** 월 적정 생활비(기본 350만 원) × 4% 룰(25배) 적용 = 은퇴 자금 자동 계산 
- 현재 자산과 매월 저축 여력을 입력받아 **예상 은퇴 날짜(D-Day)** 카운트다운 타이머 제공 (화면 정중앙 메인 컴포넌트)

### 3.3. 인터랙티브 '지출 스와이프' 시뮬레이션
- 슬라이더 바를 좌우로 움직여 이번 달 추가 지출/저축을 조절
- 조절 시 **"오마카세 비용(15만 원) 절약 시 은퇴일이 4일 앞당겨집니다!"** 와 같은 다이나믹한 문구 및 애니메이션 피드백 출력

### 3.4. 심화 기능 (리텐션 및 액션 유도)
- **결제 참기 버튼:** 충동구매 위기를 극복했을 때 누르는 액션 버튼. 화려한 애니메이션과 함께 가상 적립금 기록
- **멘탈 타격 & 힐링 알림:** 월요일 출근길, 금요일 퇴근길 등에 남은 은퇴일을 활용한 친근한 푸시 알림
- **스트레스 테스트 탭:** 소득 감소, 지출 증가 등 여러 현실적인 위기 상황을 대입해보는 시나리오 기능

## 5. 데이터 모델 및 상태 관리

### 5.1. 사용자 입력 항목 (UserProfile)
| 필드 | 타입 | 기본값 | 근거 |
|------|------|--------|------|
| `currentAssets` | number | 4억 원 | 통계청 2024 가계금융복지조사 — 40대 맞벌이 순자산 중앙값 참고 |
| `monthlyIncome` | number | 800만 원 | 통계청 2024 가계동향 — 40대 맞벌이 가구 월평균 근로소득 참고 |
| `monthlyExpense` | number | 350만 원 | 통계청 2024 가계동향 — 2인 가구(자녀 무) 평균 소비지출 참고 |
| `investReturnRate` | number | 6% | S&P 500 장기(20년) 실질 평균 수익률 기반 보수적 추정 |
| `targetExpense` | number | 350만 원 | 은퇴 후 월 생활비 (사용자 조정 가능) |

> **핵심 공식:** 목표 자금 = `targetExpense` × 12개월 × 25배 (4% 안전인출율)
> **월 저축액:** `monthlyIncome` - `monthlyExpense`
> **도달 시점:** 복리 계산으로 현재 자산이 목표 자금에 도달하는 개월 수 산출

### 5.2. 결제 참기 기록 (ResistRecord)
| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | 고유 ID |
| `amount` | number | 참은 금액 (원) |
| `category` | string | 카테고리 (외식, 쇼핑, 구독 등) |
| `savedDays` | number | 이 금액으로 앞당겨진 은퇴 일수 |
| `createdAt` | Date | 기록 시점 |

### 5.3. 저장 전략
| 단계 | 방식 | 시점 |
|------|------|------|
| **MVP (1차)** | `localStorage` | 로그인 없이 즉시 사용 가능 |
| **2차** | Supabase 또는 Firebase Firestore | 소셜 로그인 연동 후 클라우드 동기화 |

## 7. 지출 스와이프 — 항목별 매칭 예시
슬라이더 조작 시 금액 구간에 따라 실감나는 피드백 문구를 랜덤 제공한다.

| 절약 금액 | 매칭 항목 예시 | 피드백 문구 |
|-----------|---------------|-------------|
| 5만 원 | 커피 20잔 | "아아 대신 물 마시면 은퇴가 X일 빨라집니다!" |
| 10만 원 | 배달 8회 | "치킨 참으면 사표가 X일 가까워집니다!" |
| 15만 원 | 오마카세 1회 | "오마카세 포기 = 은퇴 X일 앞당김!" |
| 30만 원 | 골프 라운딩 1회 | "필드 대신 퇴직금 필드를 채우세요!" |
| 50만 원 | 해외여행 적금 | "올해 여행은 퇴직 후 세계일주로!" |

## 8. 스트레스 테스트 시나리오 정의
사용자가 현실적인 위기 상황을 대입하여 은퇴일 변동을 확인하는 기능.

| 시나리오 | 변수 변경 | 설명 |
|----------|-----------|------|
| 🔻 소득 30% 감소 | `monthlyIncome` × 0.7 | 권고사직, 이직, 연봉 삭감 |
| 🔻 투자 수익률 하락 | `investReturnRate` → 3% | 장기 침체, 하락장 |
| 🔺 생활비 50만 원 증가 | `monthlyExpense` + 50만 | 의료비, 부모 부양, 반려동물 등 |
| 🔺 목표 생활비 상향 | `targetExpense` + 100만 | 은퇴 후 더 여유로운 생활 원할 때 |
| 🎉 보너스 2000만 원 | `currentAssets` + 2000만 | 성과급, 부동산 매각 등 일시적 유입 |
| 🔻 복합 위기 | 소득↓ + 지출↑ + 수익률↓ | 최악의 시나리오 대비 |

## 10. 모바일 대응 전략
- **Mobile First:** 모든 UI를 `max-w-md` (448px) 기준으로 설계 후 확장
- **반응형 브레이크포인트:** sm(640px), md(768px) 기준으로 패딩/폰트 조정
- **PWA (Phase 3):** `next-pwa` 활용, 홈 화면 설치 유도 배너
- **터치 최적화:** 버튼 최소 44px, 슬라이더 터치 영역 확보
